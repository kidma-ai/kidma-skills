# Kidma Lesson Plan — DOCX Formatter

This skill takes structured lesson plan content and produces a branded .docx file matching Kidma's official template.

## Prerequisites

Before using this skill, always also read:
- `the docx public skill` — for general docx creation and validation best practices
- `kidma-plugins:brand skill` — for Kidma's official colors, typography, logo variants, and usage rules. Apply these to any customization or manual edits beyond the template defaults.

## How It Works

The skill includes:
- `assets/template.docx` — Kidma's official lesson plan template with branded headers, footers, logos, fonts, and styles
- `scripts/build_lesson.py` — Python script that injects content into the template

## Workflow

### Step 1: Get the Content

The content can come from:

**Option A — From kidma-lesson-generator skill:** A `lesson_content.json` file at `/home/claude/lesson_content.json` (generated by the companion skill).

**Option B — From conversation context:** If the user provides lesson content in the conversation (or you already have it), create the JSON file yourself following the schema below.

**Option C — From user-uploaded JSON:** The user uploads a JSON file directly.

### Step 2: Validate / Create the JSON

The JSON must follow the schema at `../schemas/lesson-content.schema.json`. Top-level structure:

```json
{
  "lesson_number": "1",
  "title": "נושא השיעור",
  "objectives": { "understanding": [...], "experimentation": [...], "creation": [...] },
  "materials": ["מחשב עם אינטרנט", "מצגת שיעור"],
  "learning_materials": "Free text with instructor self-study links",
  "structure": {
    "introduction":    { "description": "...", "bullets": [{"label": "לעורר עניין:", "text": "..."}] },
    "understanding":   { "description": "...", "bullets": [{"label": "ללמד:", "text": "..."}] },
    "experimentation": { "description": "...", "bullets": [{"label": "להדגים:", "text": "..."}] },
    "creation":        { "description": "...", "bullets": [{"label": "הגדרת המשימה:", "text": "..."}] },
    "summary":         { "description": "...", "bullets": [{"label": "סיכום:", "text": "..."}] }
  },
  "dead_time": "Filler activities description",
  "assessment": "How to evaluate lesson success",
  "instructor_notes": "Tips and guidance for the instructor"
}
```

All text values should be in Hebrew.

### Step 3: Run the Build Script

```bash
# Copy the template to the working directory
cp ${CLAUDE_PLUGIN_ROOT}/assets/template.docx /home/claude/template.docx

# Run the build script
python ${CLAUDE_PLUGIN_ROOT}/scripts/build_lesson.py \
  /home/claude/template.docx \
  /home/claude/lesson_content.json \
  /home/claude/lesson_plan.docx
```

### Step 4: Validate the Output

```bash
python /mnt/skills/public/docx/scripts/office/validate.py /home/claude/lesson_plan.docx
```

If validation fails, unpack, fix the XML, and repack per the docx skill instructions.

### Step 5: Deliver

```bash
cp /home/claude/lesson_plan.docx /mnt/user-data/outputs/lesson_plan.docx
```

Then use `present_files` to share with the user.

## Template Details

The template preserves these branded elements (all handled automatically by the script):

### Branding
- **Header:** Kidma logo (right) + Gefen/MoE approval badge (left) on every page
- **Footer:** Colorful wave design image + centered page number
- **Fonts & Colors:** See `kidma-plugins:kidma-brand-guidelines skill`
- **Direction:** Full RTL (right-to-left) — `bidi=1` on all paragraphs

### Style Mapping
| Template Style | Used For |
|---|---|
| `Title` | Main lesson title |
| `Subtitle` | Section headers (מטרות, עזרים, מבנה, הערכה, etc.) |
| `Heading4` | Phase headers + objective sub-headers |
| `Heading6` | Phase descriptions / instruction text |
| Normal | Regular body text |
| Bullet (numId=1) | List items |

### Page Setup
- A4 paper (11909 × 16834 twips)
- Margins: top 1700, bottom 2834 (extra room for wave footer), left/right 1440
- Header height: 567, Footer height: 1134

## Customization

### Adding Extra Sections
If you need to add sections not in the standard template (e.g., additional phases, extra bullet groups), you can edit the JSON and modify the `build_lesson.py` script. The helper functions available are:

- `title_para(number, topic)` — Title paragraph
- `subtitle(text)` — Section header
- `heading4(text)` — Phase header (dark)
- `heading4_blue(text)` — Sub-header (blue, for objectives)
- `heading6(text)` — Description text (gray italic)
- `normal_para(text)` — Regular paragraph
- `simple_bullet(text, num_id)` — Plain bullet
- `bullet_para(label, text, num_id)` — Bullet with bold label
- `empty_para()` — Spacer

### Manual Edits After Generation
For fine-tuning, you can unpack the generated .docx, edit the XML, and repack:

```bash
python /mnt/skills/public/docx/scripts/office/unpack.py lesson_plan.docx unpacked/
# Edit unpacked/word/document.xml as needed
python /mnt/skills/public/docx/scripts/office/pack.py unpacked/ lesson_plan_final.docx --original lesson_plan.docx
```
